# Bug Agent 启动和使用教程

## 一、环境准备

### 1.1 安装 Python 3.10+

```bash
# macOS
brew install python3

# Ubuntu/Debian
sudo apt install python3.10

# Windows
# 从 https://www.python.org/downloads/ 下载安装
```

验证安装：
```bash
python3 --version
```

### 1.2 安装 Java (JetBrains 插件构建需要)

```bash
# macOS
brew install openjdk@17

# 验证
java -version
```

## 二、配置步骤

### 2.1 复制环境变量文件

```bash
cd /Users/bz/code/bug-agent
cp .env.example .env
```

### 2.2 编辑 .env 文件

填入你的 API Key：

```bash
# 至少选择其中一个
DEEPSEEK_API_KEY=your-deepseek-key
# GLM_API_KEY=your-glm-key
# MINIMAX_API_KEY=your-minimax-key
```

### 2.3 修改 config.yaml

编辑 `/Users/bz/code/bug-agent/config.yaml`：

```yaml
# 日志 API 配置
log:
  api:
    base_url: "http://your-log-api-host:port"  # 改为你的日志服务地址
    api_key: "${LOG_API_KEY}"

# 项目配置 - 改为你的项目路径
project:
  root_dir: "/Users/bz/code/your-project"
  java_package_base: "com.your.company"
```

## 四、使用方法

### 4.1 API 调用方式

#### 分析错误

```bash
curl -X POST http://127.0.0.1:8765/analyze \
  -H "Content-Type: application/json" \
  -d '{
    "error_content": "java.lang.NullPointerException: Cannot invoke method on null\n\tat com.bank.loan.service.UserService.getUser(UserService.java:45)",
    "auto_fix": false
  }'
```

**返回示例：**
```json
{
  "error_type": "NULL_POINTER",
  "error_message": "Cannot invoke method on null",
  "location": {
    "file": "/Users/bz/code/loan-core-sys/src/main/java/com/bank/loan/service/UserService.java",
    "line": 45
  },
  "root_cause": "数据库查询结果未做空值校验，当查询结果为空时直接调用对象方法导致空指针异常",
  "fix_suggestion": "在调用方法前添加空值判断，或使用 Optional 处理可能为空的返回值",
  "fix_code": "if (user == null) {\n    throw new BusinessException(\"用户不存在\");\n}",
  "confidence": 0.85,
  "code_context": "..."
}
```

#### 获取日志列表

```bash
curl http://127.0.0.1:8765/logs?limit=10
```

#### 获取单条日志

```bash
curl http://127.0.0.1:8765/logs/log_123
```

#### 应用修复

```bash
curl -X POST http://127.0.0.1:8765/fix \
  -H "Content-Type: application/json" \
  -d '{
    "file_path": "/Users/bz/code/loan-core-sys/src/main/java/com/bank/loan/service/UserService.java",
    "line_number": 45,
    "fix_code": "if (user == null) {\n    throw new BusinessException(\"用户不存在\");\n}"
  }'
```

### 4.2 JetBrains 插件方式

#### 构建插件

```bash
cd /Users/bz/code/bug-agent/jetbrains-plugin
./gradlew build
```

构建完成后，插件包在 `build/distributions/` 目录

#### 安装插件

1. 打开 IntelliJ IDEA
2. File → Settings → Plugins
3. 点击齿轮图标 → Install Plugin from Disk
4. 选择生成的 `.zip` 文件
5. 重启 IDE

#### 使用插件

1. 重启后，在右侧找到 **Bug Agent** 工具窗口
2. 点击按钮获取测试环境日志
3. 在输入框粘贴错误堆栈
4. 点击"分析错误"
5. 查看分析结果
6. 如需修复，点击"应用修复"

## 五、功能说明

### 5.1 错误分析

支持解析的 Java 异常类型：
- NullPointerException
- IllegalArgumentException
- IllegalStateException
- ArrayIndexOutOfBoundsException
- ClassCastException
- ConcurrentModificationException
- SQLException
- TimeoutException
- OutOfMemoryError
- StackOverflowError
- 其他所有 Java 异常

### 5.2 代码定位

- 根据堆栈中的类名定位源文件
- 加载错误位置的代码上下文（上下10行）
- 提取相关的方法和类信息

### 5.3 LLM 分析

- 调用 DeepSeek / GLM / MiniMax 进行根因分析
- 生成修复建议和代码
- 计算置信度

### 5.4 自动修复

- 一键应用修复代码
- 生成 patch 文件

## 七、常见问题

### Q1: 启动失败，提示 "配置文件不存在"

A: 确保在 `/Users/bz/code/bug-agent` 目录下，且 `config.yaml` 文件存在

### Q2: LLM 调用失败

A: 检查 `.env` 文件中的 API Key 是否正确填入

### Q3: 代码定位失败

A: 检查 `config.yaml` 中 `project.root_dir` 是否指向正确的项目路径

### Q4: 日志获取失败

A: 检查 `config.yaml` 中 `log.api.base_url` 是否正确，以及日志服务是否可访问

### Q5: JetBrains 插件构建失败

A: 确保已安装 Java 17，并设置 `JAVA_HOME` 环境变量

## 八、配置参考

完整的 `config.yaml` 配置项说明：

```yaml
# 日志获取配置
log:
  api:
    base_url: "http://localhost:8080"    # 日志服务地址
    api_key: "${LOG_API_KEY}"             # API Key (从环境变量读取)
    endpoints:
      log_list: "/api/v1/logs"           # 日志列表接口
      log_detail: "/api/v1/logs/{log_id}" # 日志详情接口
      search: "/api/v1/logs/search"       # 搜索接口
    timeout: 30                          # 超时时间(秒)

# LLM 配置
llm:
  provider: "deepseek"                   # 当前使用的 LLM
  
  providers:
    deepseek:
      api_key: "${DEEPSEEK_API_KEY}"
      base_url: "https://api.deepseek.com/v1"
      model: "deepseek-coder"
      
    glm:
      api_key: "${GLM_API_KEY}"
      base_url: "https://open.bigmodel.cn/api/paas/v4"
      model: "glm-4"
      
    minimax:
      api_key: "${MINIMAX_API_KEY}"
      base_url: "https://api.minimax.chat/v1"
      model: "MiniMax-M2.1"

# 项目配置
project:
  root_dir: "/Users/bz/code/loan-core-sys"  # 项目根目录
  java_package_base: "com.bank.loan"       # Java 包前缀
  source_dirs:
    - "src/main/java"                      # 源代码目录
  exclude_dirs:
    - "target"
    - "build"
    - ".git"

# 服务配置
server:
  host: "127.0.0.1"                        # 监听地址
  port: 8765                               # 监听端口
  cors_origins:                            # CORS 允许的源
    - "http://localhost"
    - "http://127.0.0.1"
```
