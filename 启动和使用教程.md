## 四、使用方法

### 4.1 异常类 Bug 分析（有错误堆栈）

#### 4.1.1 基础分析（从文本输入）

```bash
curl -X POST http://127.0.0.1:8765/analyze \
  -H "Content-Type: application/json" \
  -d '{
    "error_content": "java.lang.NullPointerException: Cannot invoke method on null\n\tat com.bank.loan.service.UserService.getUser(UserService.java:45)"
  }'
```

#### 4.1.2 从文件分析

```bash
curl -X POST http://127.0.0.1:8765/analyze/file \
  -H "Content-Type: application/json" \
  -d '{
    "file_path": "/Users/bz/code/logs/error.log",
    "max_lines": 1000
  }'
```

#### 4.1.3 增强分析（完整代码走读）

```bash
curl -X POST http://127.0.0.1:8765/analyze/enhanced \
  -H "Content-Type: application/json" \
  -d '{
    "file_path": "/Users/bz/code/loan-core-sys/src/main/java/com/bank/loan/service/UserService.java",
    "line_number": 45,
    "error_type": "NULL_POINTER",
    "error_message": "Cannot invoke method on null"
  }'
```

---

### 4.2 业务类 Bug 分析（无异常，结果不符合预期）

#### 4.2.1 交互式分析（推荐）

通过问答方式收集信息，更准确地定位问题：

```bash
# 1. 开始交互式分析
curl -X POST http://127.0.0.1:8765/analyze/interactive/start

# 返回：
{
  "session_id": "a1b2c3d4",
  "status": "started",
  "question": {
    "key": "bug_description",
    "question": "请描述一下这个 bug 的具体情况？"
  },
  "progress": "1/6"
}

# 2. 回答问题
curl -X POST "http://127.0.0.1:8765/analyze/interactive/answer?session_id=a1b2c3d4&answer=提前还款时最后一期金额计算错误"

# 3. 继续回答问题直到完成...
curl -X POST "http://127.0.0.1:8765/analyze/interactive/answer?session_id=a1b2c3d4&answer=最后一期应还1000元"
curl -X POST "http://127.0.0.1:8765/analyze/interactive/answer?session_id=a1b2c3d4&answer=实际显示2000元，多算了利息"

# 4. 获取分析结果
curl http://127.0.0.1:8765/analyze/interactive/status/a1b2c3d4
```

#### 4.2.2 业务流程分析

```bash
# 分析代码是否符合业务流程描述
curl -X POST http://127.0.0.1:8765/analyze/workflow \
  -H "Content-Type: application/json" \
  -d '{
    "workflow_description": "用户还款后应该先冲抵罚息，再冲抵利息，最后冲抵本金",
    "code": "public void repay(BigDecimal amount) {\n    // 实际代码...\n}",
    "language": "java"
  }'
```

#### 4.2.3 对比预期和实际代码

```bash
curl -X POST http://127.0.0.1:8765/analyze/workflow/compare \
  -H "Content-Type: application/json" \
  -d '{
    "expected_workflow": "1. 查询用户 2. 校验权限 3. 计算金额 4. 执行操作",
    "actual_code": "实际代码...",
    "language": "java"
  }'
```

## 五、API 接口文档

### 5.1 异常分析

| 接口 | 方法 | 说明 |
|------|------|------|
| /analyze | POST | 基础错误分析 |

| /analyze/file | POST | 从文件分析错误 |
| /analyze/text | POST | 从文本分析错误 |

### 5.2 业务分析

| 接口 | 方法 | 说明 |
|------|------|------|
| /analyze/interactive/start | POST | 开始交互式分析 |

| /analyze/interactive/status/{id} | GET | 获取分析状态 |
| /analyze/workflow | POST | 业务流程分析 |
| /analyze/workflow/compare | POST | 对比预期和实际 |
| /analyze/rules/extract | POST | 提取业务规则 |
| /analyze/rules/match | POST | 检查代码符合规则 |

### 5.3 文档检索

| 接口 | 方法 | 说明 |
|------|------|------|
| /docs/search | GET | 搜索需求文档 |

| /enhanced/context | GET | 获取增强上下文 |

### 5.5 日志

| 接口 | 方法 | 说明 |
|------|------|------|
| /logs | GET | 获取日志列表 |

### 5.6 修复

## 六、支持的错误类型

### 6.1 异常类 Bug

| 类型 | 说明 |
|------|------|
| CODE_DEFECT | 代码缺陷（空指针、数组越界等） |
| UPSTREAM_PARAM | 上游参数问题 |
| UPSTREAM_DATA | 上游数据问题 |
| DB_QUERY | 数据库查询问题 |
| DB_DATA | 数据库数据问题 |
| DOWNSTREAM_CALL | 下游调用失败 |
| DOWNSTREAM_RETURN | 下游返回数据问题 |

### 6.2 业务类 Bug

通过交互式分析和业务流程分析，支持：
- 结果不符合预期
- 计算逻辑错误
- 流程顺序错误
- 业务规则未执行

## 八、常见问题

### Q1: 启动失败，提示 "配置文件不存在"

A: 确保在 `/Users/bz/code/bug-agent` 目录下，且 `config.yaml` 文件存在

### Q2: LLM 调用失败

A: 检查 `.env` 文件中的 API Key 是否正确填入

### Q3: 代码定位失败

A: 检查 `config.yaml` 中 `project.root_dir` 是否指向正确的项目路径

### Q4: 日志获取失败

A: 检查 `config.yaml` 中 `log.api.base_url` 是否正确，以及日志服务是否可访问

### Q5: 交互式分析会话超时

A: 会话目前保存在内存中，重启服务后会丢失。可以通过 `/analyze/interactive/status/{id}` 随时查看状态。

### Q6: 文档检索找不到文档

A: 检查 `config.yaml` 中 `project.document_dirs` 是否包含你的文档目录

## 九、配置参考

完整的 `config.yaml` 配置项说明：

```yaml
# 日志获取配置
log:
  default_log_dir: "/Users/bz/code/logs"  # 默认日志目录
  api:
    base_url: "http://localhost:8080"    # 日志服务地址
    api_key: "${LOG_API_KEY}"             # API Key (从环境变量读取)
    endpoints:
      log_list: "/api/v1/logs"           # 日志列表接口
      log_detail: "/api/v1/logs/{log_id}" # 日志详情接口
      search: "/api/v1/logs/search"       # 搜索接口
    timeout: 30                          # 超时时间(秒)

# LLM 配置
llm:
  provider: "deepseek"                   # 当前使用的 LLM
  
  providers:
    deepseek:
      api_key: "${DEEPSEEK_API_KEY}"
      base_url: "https://api.deepseek.com/v1"
      model: "deepseek-coder"
      
    glm:
      api_key: "${GLM_API_KEY}"
      base_url: "https://open.bigmodel.cn/api/paas/v4"
      model: "glm-4"
      
    minimax:
      api_key: "${MINIMAX_API_KEY}"
      base_url: "https://api.minimax.chat/v1"
      model: "MiniMax-M2.1"

# 项目配置
project:
  root_dir: "/Users/bz/code/loan-core-sys"  # 项目根目录
  java_package_base: "com.bank.loan"       # Java 包前缀
  source_dirs:
    - "src/main/java"                      # 源代码目录
  document_dirs:                           # 文档目录
    - "docs"
    - "doc"
    - "."
  exclude_dirs:
    - "target"
    - "build"
    - ".git"

# 服务配置
server:
  host: "127.0.0.1"                        # 监听地址
  port: 8765                               # 监听端口
  cors_origins:                            # CORS 允许的源
    - "http://localhost"
    - "http://127.0.0.1"
```
